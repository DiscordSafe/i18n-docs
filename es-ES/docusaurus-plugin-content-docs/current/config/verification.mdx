Establezca la verificación de su servidor con uno de **nuestros poderosos modos** para evitar usuarios conflictivos, multicuentas o bots de spam.

* Usando el comando `d.verification` podrá ver la configuración actual de la verificación.
* Para desactivarlo use:  `d.verification disable`

## Establecer el modo {#set-mode}
Establezca el modo de verificación que más le convenga usando `d.verification setmode <Modo>`.
Los modos de verificación son los siguientes:

| 🔧 Modo           | 📣 Descripción                                                                  | 📑 Requiere                                       | 🔐 Nivel protección |
| :---------------: | :------------------------------------------------------------------------------ | :------------------------------------------------ | :---- |
| **Approval**      | El usuario tiene que ser aprobado por un administrador con `d.approve @Usuario` | Canal, rol no nerificado, rol usuario.            | **4** |
| **Captcha**       | El usuario tiene que completar correctamente un captcha.                        | Canal, rol no nerificado, rol usuario.            | **3** |
| **Reaction**      | El usuario tiene que reaccionar a un mensaje para acceder al servidor.          | Canal, rol no verificado, rol usuario, mensaje.   | **2** |
| **Verification**  | El usuario tiene que usar el comando `d.verifyme` para entrar.                  | Canal, rol no nerificado, rol usuario.            | **1** |

## Establecer el canal {#set-channel}
Debe establecer el canal de verificación donde se mandará el captcha, o donde esté el mensaje al que tengan que reaccionar o donde tienen que poner los comandos.

Establezca el canal con `d.verification setchannel #Canal`.

## Establecer el rol de no verificado {#set-unverified}
:::caution Aviso
Usted tendrá que configurar este rol en su servidor para que solo vea los canales que usted quiera y que cuando se verifique tenga acceso completo al servidor.
:::

Este será el rol que se les dará a los nuevos usuarios al entrar al servidor antes de verificarse.

Establezca el rol de no verificados con:
`d.verification setguest @Rol`.

## Establecer el rol de Usuario {#set-verified}
Este será el rol que se les dará a los nuevos usuarios al verificarse (normalmente el rol Usuario).

El comando para establecerlo es `d.verification setuser @Rol`.

## Establecer el mensaje {#set-message}
:::caution Aviso
**Solo es necesario este ajuste** en el modo **reaction**.
:::

Establezca el mensaje al que tendrán que reaccionar los no verificados al unirse a su servidor.
**Importante,** el mensaje que establezca debe de estar dentro del canal que configuró anteriormente.

Configure el mensaje usando: `d.verification setmessage MensajeID`.

## Establecer el mínimo de edad {#set-min-age}
Establezca el **mínimo de días que deben tener las cuentas** cuando se unen a su servidor, así evita que entren multicuentas o auto-bots de spam.

* Configure el mínimo con: `d.verification setage <1-30 Días>`.
* Puede desactivarlo con `d.verification setage disable`.

## Establecer máximo de intentos {#set-max-try}
:::caution Aviso
**Este es un ajuste opcional** del modo **captcha**.
:::

Establezca el **número máximo de intentos** que pueden hacer los usuarios en el **captcha**.

Configure el máximo de intentos con ``d.verification setattempts [Número]``.

Puede volver a la cantidad por defecto con ``d.verification setattempts default``.

## Establecer máximo de minutos {#set-max-min}
:::caution Aviso
**Este es un ajuste opcional** del modo **captcha**.
:::

Establezca el **tiempo máximo de espera** en la verificación **captcha**.

Configure el máximo de intentos con ``d.verification settime [Número]`` (en minutos).

Puede volver a la cantidad por defecto con ``d.verification settime default``.
